Macの環境整備

Dec 22nd, 2013

factorやる

もう一回factorやるかー、簡単なプログラム書いてみよ。 と思ったら、新しいPCに何も入ってなかったので、まずは環境設定。

エディタ起動

とりあえず、Emacsをおもむろに落とす。 gistに貼った古いemacs.elをコピペし、.emacs.d/init.elに置いて起動。古いが、ざっくり変えるので、とりあえずキーバインドとかがまともに効けばなんでもいい。 昔Carbon Emacsの時代に設定していたmac-add-ignore-shortcutがもう使えないので、それが原因で立ち上がらない。その部分はとりあえず消した。 ただ、それがなくなったのでControll-SpaceがMacのSpotlightにとられてしまう。ので、Spotlight側の設定を変更した（システム環境設定->Spotlightからショートカットを全部切る。使わないので）。

github設定

エディタの設定記事とかをブログに書き残しておこうと思ったんだが、このブログはgithubでホスティングされていて、そもそもgithubにアクセスするためのキーとか色々と無くて設定した。 前のPCからキーを持ってきて設定しようとしたが、うまくいかず、最終的にはもう一個キーを追加。どこのPCのどのアカウントで作ったかをキーのtitleとして置いておけば、たぶん困ることはないので、次から新しく作るようにしよう。

git取得

そもそもその2として、そもそもgitがない。Command Line Toolsをダウンロードして使えるようにした。

Octopress設定

ブログをgit cloneして落としてきたんだが、ブログを生成しているOctopress自体は実はリポジトリに含まれていない。なので、別の環境で単にblogをgit cloneしただけだと、preview機能はおろか何もできない。 ということでOctopressを落としてきたんだが、bundle installすると素直に通らない。 xcode-select: Error: No Xcode is selected. Use xcode-select -switch <path-to-xcode>, or see the xcode-select manpage (man xcode-select) for further information. 調べると情報が出てくるが、Xcodeを入れずにCommand Line Toolsだけ入れるとこういう悲しいことになるらしい。回避方法あるらしいが、面倒になってきたのでXcodeをダウンロード。2GB以上もあるので、ダウンロードの間にこれを書いている。 あとrubyが1.8というのもさすがにないわーなので、rbenvで2.0に上げておいた。 http://qiita.com/skinoshita/items/2cf0c27a6ff4f0602568 しかしこのrubyバージョンアップで数時間が経過、吐きかけるのであった…。 Xcodeを入れて再度bundle installすると、 Gem::InstallError: liquid requires RubyGems version >= 1.3.7. Try ‘gem update –system’ to update RubyGems itself. An error occurred while installing liquid (2.3.0), and Bundler cannot continue. Make sure that gem install liquid -v '2.3.0' succeeds before bundling. とか出てしまう。言われたとおり、普通にgemを上げてもだめ。 gem install liquid -v ‘2.3.0’ は通る。rehashしてるし、gemもbundlerもちゃんとバージョンあがっている。 結論としては、rubyは2.0に上がったんだが、bundle installした時に見てるgemが何故かsystem(1.8のruby)という事態。意味不明なのでとりあえず1.8に戻してgemをupdateしたら通った。疲れたのでこれ以上は追わないが、budlerを入れたときのrubyのversionがsystemだったとかじゃねぇの。 とはいえ1.8のまま使うとかはないので、2.0に切り替えて、bundlerを入れ直したりした。

ブログ設定

既にblogのコンテンツはgithubに置いてあるので、こいつをOctopressに置いて、動くようにする。この方法をドキュメントとか見ながら探したんだけど、よくわからなかった。 結局、 cd octpress rm -rf source # とりあえず今あるsourceを全部消す

自分のブログのテーマプラグインを設定する（https://github.com/roberto/oct2-orange）

git clone git@github.com:syokai/syokai.github.com.git contents cp -R contents source cp 元の環境のconfig.yml config.yml # なければ頑張って書く rm -R deploy cp -R source/ deploy # 最終的にdeployを上げているっぽい気がしたので書き換えておいた とかしてみた。かなりテキトーだが、分からんのでしょうがない。 とりあえずこれで、 rake generate rake preview で見れたし、 rake gen_deploy したらgithubにも上がったようなので問題ないかと思う（このコミットは微妙なスタイル変更になったんだが何故なんだろう…）。 思うに、まじめにoctopressを使ってる人は、octopressディレクトリ全体も自分のgithubに置いたりするんだろう。pushして公開できたのを見て、「ワーイ、バックアップとれたよー」とか思っていると、こういう大変なことに巻き込まれる。

エディタ設定

ブログはさておいて、本題のエディタを設定する。

KeyRemap4MacBook

英数を単独で押した時には英数で、他のキーと併用した時はcommand(L)で、というキー配列にするためにKeyRemap4MacBookを使用する。キー配列をEmacsの外で変える（例えばキー配列の変更ソフトなどを利用する）のは正直好きじゃないのだが、しょうがない。elispで設定できるならそうしたいが知らない。

.el

とりあえず、仕事とか全部忘れてプログラミング言語系の設定は全部削除。あとtrampとかそういうのも全部さよならした。 残ったのはキーバインドとかそういうもののみ。50行ぐらいのミニサイズになった。ここから追加していく。

package

公式ではないmelpaにしかないパッケージもあるので、以下を登録しておく。 (require ‘package) (add-to-list ‘package-archives ‘(“melpa” . “http://melpa.milkbox.net/packages/”)) (package-initialize)

helm

時代はanything.elではなくhelmらしい。helmはanythingのfork。 melpa入れておけば、packageで入れられる。 設定はこちらを参考にした。 http://yoshiori.github.io/blog/2013/01/30/dot-emacs-renewal/

fuel

emacsでfactorを書くサポートをしてくれるelisp。これを入れるための長い戦いだった…。 fuelはfactorの中にREADMEがあるので読むと良いと思う。疲れ果てて本題を書けない本末転倒っぷりを晒して終了。
